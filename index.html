个人博客搭建Hexo+GitHub

发表于 2017-09-03

#前言部分
本文主要参考：知乎上吴润的GitHub+Hexo搭建个人网站详细教程（https://zhuanlan.zhihu.com/p/26625249）和简书上代码咖啡的20分钟教你使用hexo搭建github博客（http://www.jianshu.com/p/e99ed60390a8）在以上基础上简单修改和调整搭建顺序完成。

#正文部分
首先实现本地预览，接着实现github预览，最后实现域名预览。
1.github创建个人仓库。
首先要有github个人账号，然后创建一个新的仓库new repository,仓库名为用户名.github.io比如cao.github.io。
2.安装Git
Git是开源的分布式版本控制系统，用于敏捷高效地处理项目。Git可以将本地搭建的网站同步到GitHub上。安装完成可以在命令行里输入git测试是否安装成功，若安装成功，将你的Git与github账号绑定。在桌面上，鼠标右键打开Git Bash。设置user.name和user.email配置信息：
git config –global user.name “你的GitHub用户名”
git config –global user.email “你的GitHub注册邮箱”
生成ssh密匙文件：
ssh-keygen -t rsa -C “你的GitHub注册邮箱”
然后直接三个回车即可，默认不需要设置密码
然后找到生成的.ssh的文件夹中的id_rsa.pub密钥，将内容全部复制（可以用记事本打开）。
打开GitHub_Settings_keys页面，新建new SSH Key。
其中Title是标题，任意填写即可。
Key是密钥，将刚刚复制的id_rsa.pub内容粘贴进去，最后点击Add SSH key。
可以在Git Bash中检测GitHub公钥设置是否成功，输入ssh git@github.com若出现PTY allocation request failedd on channel 0 Hi 用户名！ You ‘ve successfully authenticated, but GitHub does not provide shell access.Connection to github.com closed.如上则说明设置成功。这里之所以设置GitHub密钥原因是，通过非对称加密的公钥与私钥来完成加密，公钥放置在GitHub上，私钥放置在自己的电脑里。GitHub要求每次推送代码都是合法用户，所以每次推送都要输入米账号密码验证推送用户是否是合法用户，为了省去每次输入密码的步骤，采用了ssh,当你推送的时候，git就会匹配你的私钥跟GitHub上面的公钥是否是配对的，若是匹配就认为你是合法用户，则允许推送。这样可以保证每次的推送都是正确合法的。

3.安装Node.js

Hexo是基于Node.js,注意安装Node.js会包含环境变量及npm的安装，安装后，检测Node.js是否安装成功，在命令行中输入 node -v 出项版本号则说明安装成功。检测npm是否安装成功，在命令行中输入npm -v出现版本号，则说明npm安装成功。至此，安装Hexo的环境已经全部搭建完成。

4.安装Hexo
Hexo就是我们个人博客网站的框架，这里需要自己在电脑里创建一个文件夹，可以命名为Blog,Hexo框架与以后你自己发布的网页都在这个文件夹里。创建好后，进入Blog文件夹中，按住shift键，右击鼠标，点击命令行，使用npm命令安装Hexo,输入 npm install -g hexo-cli
这个安装时间较长耐心等待。安装好hexo以后，可以输入hexo来检测hexo是否安装成功。

5.初始化博客
hexo init blog //这里是在Blog下建立了个blog文件夹
cd blog //进入到blog文件夹中或者直接在blog文件夹里shift+右键点击
npm install //node.js的命令，根据博客既定的dependencies配置安装所有的依赖包

6.在本地发布一篇博客
hexo new “first”
在本地博客文件夹source->_posts文件夹下看到我们新建的markdown文件。可以用Sublime Text 3打开（其中sublime text 3要进行markdown配置）。在里面写下，你好，欢迎来到问哦的个人技术博客。保存后我们可以进行本地发布。
hexo sersver
打开浏览器，输入http://localhost:4000/
我们就可以在浏览器端看到我们搭建好的博客和发布的文章。

7.使用github仓库名进行访问
先关闭本地服务器ctrl+C,将hexo与github关联起来。在blog根目录里有个_config.yml站点配置文件，用记事本打开，翻到最后修改为：
deploy:
type: git
repo: 在github上创建仓库的完整路径，记得加上.git
branch: master
注意冒号后面一定要加上一个空格。
保存站点配置文件。
在blog命令行中输入npm install hexo-deployer-git –save
然后输入两条命令：hexo g和hexo d
等大约1min,在浏览器中输入github仓库名，就可以在网络上访问你的博客了，如cao.github.io

8.域名访问
从阿里云购买一个域名，我买的是55元/年。然后控制台里面有个解析，填写三条记录即可
从左到右依次是记录类型，主机记录，记录值。
CNAME www cao.github.io
A @ 192.30.252.153
A @ 192.20.252.154
然后进入本地博客文件夹blog/source目录下，打开sublime 创建一个新的文件，输入你的域名，不要带www,保存，保存类型为所有文件，命名为CNAME。
在blog目录中，按住shift键，右击,打开命令行，依次输入hexo g,hexo d。等大约1min，就可以域名访问了。

9.更换网站主题
可以在blog目录的themes文件夹中查看自己的主题。
在blog目录中，打开命令行输入
git clone https://github.com/iissnan/hexo-theme-next themes/next
这是将Next主题下载到blog目录的themes主题下的next文件夹中。打开站点的_config.yml配置文件，修改主题为next。找到# Extensions,theme: next
打开主题的_config.yml配置文件，找到Scheme Settings，有4个样式，选择一个自己喜欢的样式，去掉前面的#号，保存。
hexo g,hexo d。再次访问就可以看到变化了。
至于每次发布前输入hexo clean。是清理缓存，在上述中并没有写出来，因为一使用这个命令，本地的文件有一次就消失了。
